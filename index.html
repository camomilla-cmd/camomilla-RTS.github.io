App 6: Patient Monitoring Demo

This project demonstrates a small embedded patient monitoring system that reacts to changing sensor values and emergency input in real time. It uses an ESP32 running FreeRTOS to coordinate multiple tasks, display patient status on an OLED screen, and trigger visual alerts with LEDs.

Watch the Demo

Project Overview

App 6 simulates a patient assistance and monitoring device. The system reads a sensor value, checks whether it crosses a threshold, and updates outputs to help communicate the patient's status. A button interrupt also allows the system to trigger an emergency-style alert for a fall or assistance event.

The program is designed around multitasking, which means several jobs run at the same time: reading sensor data, handling alerts, updating the OLED display, and blinking a heartbeat LED to show that the device is active.

Key Features

  • Reads analog sensor input on the ESP32.
  • Uses FreeRTOS tasks for concurrent system behavior.
  • Displays patient status messages on an OLED screen.
  • Turns on a red LED during alert conditions.
  • Uses a button interrupt to simulate a patient fall or assistance event.
  • Logs system activity through serial output for debugging and observation.

How It Works

Monitoring

A task continuously reads the sensor value and sends the latest data through a queue so other parts of the system can respond without blocking one another.

Alerts

When the sensor rises above the configured threshold, the system shows a warning on the OLED display and turns on the red LED to indicate a possible problem.

Emergency Input

A button interrupt simulates a patient emergency event. When triggered, the display changes to a fall alert message and the alert LED activates.
